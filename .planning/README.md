# Planning Workspace

This directory captures project context and a practical plan for the current state of `OUFreeFood`.

Files:
- `PROJECT.md`: product context, goals, constraints, and scope boundaries.
- `STATE.md`: current implementation snapshot, architecture notes, and risks.
- `ROADMAP.md`: prioritized phases for improving the project.
- `PHASE-1-PLAN.md`: issue-sized execution plan for the first roadmap phase.

Recommended usage:
1. Read `PROJECT.md` to understand why the project exists and what it should optimize for.
2. Read `STATE.md` before changing code so you understand the actual implementation and weak spots.
3. Use `ROADMAP.md` to choose the next phase of work and keep execution focused.
